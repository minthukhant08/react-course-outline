# Optimistic Hook (useOptimistic)

`useOptimistic` ဆိုတာ UI ကို အရင်ဆုံး (optimistically) update လုပ်ပေးပြီး နောက်ကွယ်မှာ asynchronous အလုပ်တွေ (ဥပမာ - API call) ပြီးဆုံးတဲ့အထိ စောင့်ဆိုင်းပေးတဲ့ React Hook တစ်ခု ဖြစ်ပါတယ်။ ဒါဟာ user experience ကို ပိုမိုမြန်ဆန်ချောမွေ့စေပါတယ်။

---

## 1. What is Optimistic UI?

သာမန်အားဖြင့် user က button တစ်ခုနှိပ်ရင် API ကနေ ရလဒ်ပြန်လာမှ UI ကို ပြောင်းလဲလေ့ရှိပါတယ်။ ဒါကို "Pessimistic UI" လို့ ခေါ်ပါတယ်။ "Optimistic UI" ကတော့ API ရလဒ်ကို မစောင့်တော့ဘဲ UI ကို ချက်ချင်းပြောင်းလိုက်ပြီး API error တက်မှသာ မူလအတိုင်း ပြန်ပြင်ပေးတဲ့ နည်းလမ်းဖြစ်ပါတယ်။

---

## 2. Basic Usage

`useOptimistic` ကို များသောအားဖြင့် forms တွေနဲ့ actions တွေမှာ တွဲသုံးပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  options={{ editorHeight: "600px" }}
  files={{
    "/App.tsx": `import { useOptimistic, useState } from 'react';

interface Message {
  text: string;
  sending: boolean;
}

export default function MessageBox() {
  const [messages, setMessages] = useState<Message[]>([
    { text: "Hello there!", sending: false }
  ]);

  const [optimisticMessages, addOptimisticMessage] = useOptimistic(
    messages,
    (state, newMessage: string) => [
      ...state,
      { text: newMessage, sending: true }
    ]
  );

  async function sendMessage(formData: FormData) {
    const message = formData.get("message") as string;
    addOptimisticMessage(message);
    
    // Mock API call
    await new Promise(res => setTimeout(res, 1000));
    setMessages(prev => [...prev, { text: message, sending: false }]);
  }

  return (
    <div>
      <h3>Optimistic Chat</h3>
      <div style={{ border: '1px solid #ccc', padding: '10px', height: '120px', overflowY: 'auto' }}>
        {optimisticMessages.map((m, i) => (
          <div key={i} style={{ color: m.sending ? '#888' : '#000' }}>
            {m.text} {m.sending && <small>(Sending...)</small>}
          </div>
        ))}
      </div>
      <form action={sendMessage}>
        <input name="message" placeholder="Type a message..." required />
        <button type="submit">Send</button>
      </form>
    </div>
  );
}`,
  }}
/>

## 3. Error Handling and Rollback

`useOptimistic` ရဲ့ အားသာချက်ကတော့ server ဘက်က error တက်ခဲ့ရင် UI ကို မူလ state အတိုင်း အလိုအလျောက် ပြန်ပြင်ပေး (Rollback) တာပဲ ဖြစ်ပါတယ်။ 

အောက်ပါ code မှာ "Fail this message" လို့ ရေးပြီး ပို့မယ်ဆိုရင် server error တက်မှာဖြစ်ပြီး၊ ခဏအကြာမှာ optimistic message ပျောက်သွားတာကို တွေ့ရမှာပါ။

<Sandpack 
  template="react-ts"
  theme="dark"
  options={{ editorHeight: "600px" }}
  files={{
    "/App.tsx": `import { useOptimistic, useState } from 'react';

interface Task {
  id: number;
  text: string;
  sending?: boolean;
}

export default function TaskList() {
  const [tasks, setTasks] = useState<Task[]>([
    { id: 1, text: "Existing task" }
  ]);

  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask: string) => [
      ...state,
      { id: Date.now(), text: newTask, sending: true }
    ]
  );

  async function addTask(formData: FormData) {
    const text = formData.get("task") as string;
    addOptimisticTask(text);

    try {
      // Mock API call
      await new Promise(res => setTimeout(res, 1500));
      
      if (text.toLowerCase() === "fail") {
        throw new Error("Server Error!");
      }

      setTasks(prev => [...prev, { id: Date.now(), text }]);
    } catch (e) {
      alert("Failed to add task. Rolling back...");
      // Actual state (tasks) update မလုပ်တဲ့အတွက် 
      // Optimistic UI က မူလအတိုင်း အလိုအလျောက် ပြန်ဖြစ်သွားပါမယ်။
    }
  }

  return (
    <div>
      <h3>Task List (Error Simulation)</h3>
      <ul>
        {optimisticTasks.map(t => (
          <li key={t.id} style={{ opacity: t.sending ? 0.5 : 1 }}>
            {t.text} {t.sending && "(Syncing...)"}
          </li>
        ))}
      </ul>
      <form action={addTask}>
        <input name="task" placeholder="Type 'fail' to see rollback" required />
        <button type="submit">Add Task</button>
      </form>
    </div>
  );
}`,
  }}
/>

### Output
```txt
(Type 'fail' and send: The task appears temporarily, 
then an alert shows up, and the task disappears.)
```

👉 `useOptimistic` ဟာ base state (tasks) အပေါ်မှာပဲ မူတည်တာဖြစ်လို့ base state ပြောင်းလဲခြင်း မရှိရင် (Error တက်ရင်) အလိုအလျောက် rollback ဖြစ်သွားတာပါ။
