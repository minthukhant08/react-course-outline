# Zod Basics

Zod ဆိုတာ TypeScript-first schema declaration နဲ့ validation library တစ်ခု ဖြစ်ပါတယ်။ Data တွေကို runtime မှာ တိကျစွာ validate လုပ်နိုင်ဖို့အတွက် ဒီ library ကို အသုံးများကြပါတယ်။

---
## 1. Primitive Types

Zod မှာ အခြေခံကျတဲ့ JavaScript primitive data types တွေကို စစ်ဆေးဖို့အတွက် ရိုးရှင်းတဲ့ schema တွေကို အသုံးပြုနိုင်ပါတယ်။
အသုံးအများဆုံး schema တွေက `string`, `number`, `boolean` တို့ ဖြစ်ပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  customSetup={{ dependencies: { "zod": "latest" } }}
  options={{ showConsole: true, showConsoleButton: true, editorHeight: "500px" }}
  files={{
    "/App.tsx": `import { z } from "zod";

export default function App() {
  const stringSchema = z.string();
  const numberSchema = z.number();
  const boolSchema = z.boolean();

  return (
    <div>
      <p>String check ("Hi"): {JSON.stringify(stringSchema.safeParse("Hi").success)}</p>
      <p>Number check ("123"): {JSON.stringify(numberSchema.safeParse("123").success)}</p>
      <p>Boolean check (true): {JSON.stringify(boolSchema.safeParse(true).success)}</p>
    </div>
  );
}`,
  }}
/>

### Output
```txt
String check ("Hi"): true
Number check ("123"): false
Boolean check (true): true
```

👉 Primitives တွေကို သေချာစွာ စစ်ဆေးခြင်းက runtime မှာ data ကို protect နိုင်ပါတယ်။

## 2. Validations and Constraints

Schema တွေမှာ အပို Rules တွေ (Constraints) ကိုပါ ထည့်နိုင်ပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  customSetup={{ dependencies: { "zod": "latest" } }}
  options={{ showConsole: true, showConsoleButton: true, editorHeight: "500px" }}
  files={{
    "/App.tsx": `import { z } from "zod";

export default function App() {
  const passwordSchema = z.string().min(8).max(20);
  const ageSchema = z.number().min(18).max(100);

  return (
    <div>
      <p>Pass "123": {JSON.stringify(passwordSchema.safeParse("123").success)}</p>
      <p>Age 25: {JSON.stringify(ageSchema.safeParse(25).success)}</p>
    </div>
  );
}`,
  }}
/>

### Output
```txt
Pass "123": false
Age 25: true
```

## 3. parse vs safeParse

Zod မှာ data တွေကို validate လုပ်ဖို့ `parse()` နဲ့ `safeParse()` ဆိုပြီး နည်းလမ်းနှစ်မျိုးရှိပါတယ်။ 

- **parse()**: Validation အောင်မြင်ရင် data ကို return ပြန်ပေးပြီး၊ ကျရှုံးရင် error (ZodError) throw လုပ်ပါတယ်။
- **safeParse()**: Validation အောင်မြင်သည်ဖြစ်စေ၊ ကျရှုံးသည်ဖြစ်စေ object တစ်ခုကို အမြဲ return ပြန်ပေးပါတယ်။ `{ success: true, data: ... }` သို့မဟုတ် `{ success: false, error: ... }` ပုံစံမျိုး ဖြစ်ပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  customSetup={{ dependencies: { "zod": "latest" } }}
  options={{ showConsole: true, showConsoleButton: true, editorHeight: "500px" }}
  files={{
    "/App.tsx": `import { z } from "zod";

export default function App() {
  const schema = z.string();

  const handleParse = () => {
    try {
      console.log("Parsing 'hello'...");
      console.log(schema.parse("hello"));
      console.log("Parsing 123...");
      console.log(schema.parse(123)); 
    } catch (err) {
      console.log("parse() failed and threw an error");
    }
  };

  const handleSafeParse = () => {
    console.log("SafeParsing 123...");
    const result = schema.safeParse(123);
    if (result.success) {
      console.log("Success:", result.data);
    } else {
      console.log("safeParse() failed safely with success: false");
    }
  };

  return (
    <div>
      <button onClick={handleParse}>Test parse()</button>
      <button onClick={handleSafeParse}>Test safeParse()</button>
      <p>Click buttons and check the console below.</p>
    </div>
  );
}`,
  }}
/>

### Output
```txt
Parsing 'hello'...
hello
Parsing 123...
parse() failed and threw an error
SafeParsing 123...
safeParse() failed safely with success: false
```

👉 `parse()` က error throw တာကြောင့် try-catch နဲ့ သုံးသင့်ပြီး၊ `safeParse()` ကတော့ flow control အတွက် ပိုအဆင်ပြေပါတယ်။

## 4. Object Schemas

Zod ရဲ့ အသုံးအဝင်ဆုံး feature တစ်ခုကတော့ ရှုပ်ထွေးတဲ့ objects တွေကို တိကျစွာ စစ်ဆေးနိုင်တာ ဖြစ်ပါတယ်။ ဒါဟာ API ကနေ လာတဲ့ JSON data တွေကို validate လုပ်ဖို့အတွက် အကောင်းဆုံး နည်းလမ်းဖြစ်ပါတယ်။

---

## 5. Defining Objects

`z.object()` ကို အသုံးပြုပြီး object တစ်ခုမှာ ပါဝင်ရမယ့် key တွေနဲ့ သူတို့ရဲ့ data types တွေကို သတ်မှတ်နိုင်ပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  customSetup={{ dependencies: { "zod": "latest" } }}
  options={{ editorHeight: "500px" }}
  files={{
    "/App.tsx": `import { z } from "zod";

const UserSchema = z.object({
  username: z.string(),
  email: z.string().email(),
  age: z.number().optional()
});

export default function App() {
  const result = UserSchema.safeParse({
    username: "mgmg",
    email: "mgmg@gmail.com"
  });

  return (
    <div>
      <h3>Object Validation</h3>
      <p>Success: {JSON.stringify(result.success)}</p>
      {result.success && <pre>{JSON.stringify(result.data, null, 2)}</pre>}
    </div>
  );
}`,
  }}
/>

### Output
```txt
Object Validation
Success: true
{
  "username": "mgmg",
  "email": "mgmg@gmail.com"
}
```

## 6. Nested Objects

Object တစ်ခုထဲမှာ အခြား object schema တွေကို ထည့်သွင်းပြီး အဆင့်ဆင့် စစ်ဆေးနိုင်ပါတယ်။

<Sandpack 
  template="react-ts"
  theme="dark"
  customSetup={{ dependencies: { "zod": "latest" } }}
  options={{ editorHeight: "500px" }}
  files={{
    "/App.tsx": `import { z } from "zod";

const AddressSchema = z.object({
  street: z.string(),
  city: z.string()
});

const UserProfileSchema = z.object({
  name: z.string(),
  address: AddressSchema
});

export default function App() {
  const result = UserProfileSchema.safeParse({
    name: "Su Su",
    address: {
      street: "Main St",
      city: "Yangon"
    }
  });

  return (
    <div>
      <h3>Nested Object Validation</h3>
      <p>Success: {JSON.stringify(result.success)}</p>
    </div>
  );
}`,
  }}
/>

### Output
```txt
Nested Object Validation
Success: true
```

👉 Object schemas တွေကို အသုံးပြုခြင်းဖြင့် application တစ်ခုလုံးမှာ data flow မှန်ကန်မှုကို ထိန်းသိမ်းနိုင်ပါတယ်။
