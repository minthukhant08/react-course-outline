# Modules (Import/Export)

JavaScript Modules ဆိုတာ code တွေကို `file` တစ်ခုတည်းမှာ အကုန်မရေးဘဲ အစိတ်အပိုင်းလေးတွေ ခွဲပြီး ရေးသားနိုင်တဲ့ နည်းလမ်းဖြစ်ပါတယ်။

---

## 1. Why Modules?

Project တစ်ခု ကြီးမားလာတဲ့အခါ code တွေအားလုံးကို `file` တစ်ခုတည်းမှာ သိမ်းထားရင် ဖတ်ရခက်ပြီး ပြင်ဆင်ရ ခက်ခဲလာပါတယ်။ Module တွေကို သုံးခြင်းဖြင့် -
- Code တွေကို သပ်ရပ်စွာ စုစည်းနိုင်တယ် (Organization)
- တစ်နေရာမှာ ရေးထားတဲ့ code ကို တခြား`file` တွေမှာ ပြန်သုံးနိုင်တယ် (Reusability)
- Variable အမည်တွေ တစ်ခုနဲ့တစ်ခု တိုက်ဆိုင်မှုမရှိအောင် ကာကွယ်ပေးတယ် (Encapsulation)

## 2. Basic Syntax

Module စနစ်မှာ အဓိက keywords (၂) ခု ရှိပါတယ် -
- `export`: မိမိ`file` ထဲက code (variable, function, class) ကို တခြား`file` မှာ သုံးလို့ရအောင် ထုတ်ပေးခြင်း။
- `import`: တခြား`file` က ထုတ်ပေးလိုက်တဲ့ code တွေကို မိမိ`file` ထဲမှာ ယူသုံးခြင်း။

👉 Modern web development မှာ code တွေကို module ပုံစံနဲ့ ရေးသားခြင်းက standard တစ်ခု ဖြစ်ပါတယ်။
