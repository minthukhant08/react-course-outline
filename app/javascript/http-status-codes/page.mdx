# HTTP Status Codes

HTTP Status Codes ဆိုတာ client (browser) ဘက်က ပို့လိုက်တဲ့ request တစ်ခုကို server က လက်ခံရရှိပြီးတဲ့အခါ ဘယ်လိုရလဒ် ထွက်လာသလဲဆိုတာကို ပြန်လည်အကြောင်းကြားပေးတဲ့ ကိန်းဂဏန်း (၃) လုံး ကုဒ်များ ဖြစ်ပါတယ်။

---

## 1. Status Code Ranges

Status code တွေကို သူတို့ရဲ့ အဓိပ္ပာယ်ပေါ် မူတည်ပြီး အုပ်စု (၅) စု ခွဲခြားနိုင်ပါတယ် -

- **100 - 199 (Informational)**: Request ကို လက်ခံရရှိပြီး အလုပ်လုပ်နေဆဲဖြစ်ကြောင်း ပြတယ်။
- **200 - 299 (Success)**: Request ကို အောင်မြင်စွာ လက်ခံရရှိပြီး အလုပ်ပြီးမြောက်ကြောင်း ပြတယ်။ (ဥပမာ- 200 OK, 201 Created)
- **300 - 399 (Redirection)**: တခြား URL တစ်ခုသို့ ထပ်ဆင့်သွားရန် လိုအပ်ကြောင်း ပြတယ်။ (ဥပမာ- 301 Moved Permanently)
- **400 - 499 (Client Error)**: Client (User) ဘက်က အချက်အလက်မှားယွင်းမှု သို့မဟုတ် Validation မအောင်မြင်မှုများကို ပြတယ်။ (ဥပမာ- 404 Not Found, 400 Bad Request)
- **500 - 599 (Server Error)**: Server သို့မဟုတ် System ပိုင်းဆိုင်ရာ ချို့ယွင်းမှုများကို ပြတယ်။ (ဥပမာ- 500 Internal Server Error, 503 Service Unavailable)

## 2. Importance of Status Codes

Status code တွေကို မှန်ကန်စွာ နားလည်ထားခြင်းက program ထဲမှာ တက်လာတဲ့ error တွေကို ဖြေရှင်း (debug) တဲ့အခါမှာ အများကြီး အထောက်အကူ ဖြစ်စေပါတယ်။

အသေးစိတ်ကို ဒီမှာ [MDN HTTP Status Codes](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status) ဖတ်နိုင်ပါတယ်။

👉 Status code များကို မှန်ကန်စွာ အသုံးပြုခြင်းသည် client နဲ့ server ကြား ဆက်သွယ်မှုကို ပိုမိုရှင်းလင်းစေပါတယ်။
