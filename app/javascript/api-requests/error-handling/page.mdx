# Handling Errors

Network request တွေမှာ error တွေ အမျိုးမျိုးရှိနိုင်တဲ့အတွက် စနစ်တကျ ကိုင်တွယ်ဖို့ အရေးကြီးပါတယ်။

---

## 1. Checking Response Status

`fetch()` ဟာ 404 (Not Found) ဒါမှမဟုတ် 500 (Server Error) တက်ရင်တောင် reject မဖြစ်ပါဘူး။ ဒါကြောင့် `response.ok` ကို စစ်ဆေးဖို့ လိုပါတယ်။

```js
async function getData() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/invalid-url');
    
    if (!response.ok) {
      throw new Error("Data ရယူရာမှာ အဆင်မပြေပါ: " + response.status);
    }

    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.log("ဖမ်းမိတဲ့ Error:", error.message);
  }
}

getData();
```

### Output

```txt
ဖမ်းမိတဲ့ Error: Data ရယူရာမှာ အဆင်မပြေပါ: 404
```

👉 Network လုံးဝ မရှိတော့တဲ့ အခြေအနေမှာသာ `fetch()` က reject ဖြစ်မှာပါ။
